1
Open Terminal
Get your Mac's command line ready

Everything in this guide happens in the Terminal — a text interface where you type commands. Don't worry, we'll give you every command to copy-paste.

To open Terminal on your Mac:

💡 Tip

Once Terminal is open, leave it open for the whole guide. All commands run in the same window.

2
Install Homebrew
Mac's package manager — needed to get Python 3.11

Homebrew lets you install developer tools on Mac easily. Paste this entire command and hit Enter. It will ask for your Mac password.

Terminal
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

This takes 2–3 minutes. When it finishes you'll see "Installation successful!"

🔧 Already installed?

If it says Homebrew is already installed, skip to the next step.

3
Add Homebrew to PATH
Tell your terminal where to find Homebrew

After installing, Homebrew asks you to run 3 commands to make it accessible. Run all three:

Terminal — run all three
echo >> /Users/$(whoami)/.zprofile echo 'eval "$(/opt/homebrew/bin/brew shellenv zsh)"' >> /Users/$(whoami)/.zprofile eval "$(/opt/homebrew/bin/brew shellenv zsh)"
💡 Why?

Homebrew installs itself in /opt/homebrew but your terminal doesn't know to look there. These commands teach it.

4
Install Python 3.11
The eval-protocol package requires Python 3.10+

Your Mac ships with Python 3.9 which is too old. Install a newer version via Homebrew:

Terminal
brew install python@3.11

Takes about 1–2 minutes to download and install.

🔧 Common Error

If you see "pip not found" or "pip3 not found" — that's fine, use pip3.11 instead as shown in the next steps.

5
Install & Set Up eval-protocol
Fireworks' CLI tool for running RFT jobs

Install the package, clone the math quickstart repo, and write your API key:

Terminal — paste all at once
pip3.11 install eval-protocol git clone https://github.com/eval-protocol/quickstart-gsm8k.git cd quickstart-gsm8k pip3.11 install -r requirements.txt echo "FIREWORKS_API_KEY=fw_YOUR_KEY_HERE" > .env
🔑 API Key

Replace fw_YOUR_KEY_HERE with your actual key from fireworks.ai/account/api-keys. Keys start with fw_.

6
Fix the argparse Bug
A one-time patch for a bug in eval-protocol

The current version of eval-protocol has a bug where two CLI arguments share the same name. Run this patch to fix it:

Terminal — paste entire block
python3.11 - <<'EOF' import pathlib f = pathlib.Path("/opt/homebrew/lib/python3.11/site-packages/eval_protocol/cli_commands/utils.py") src = f.read_text() old = """ if unwrap_union(hint) is bool: parser.add_argument(*flags, action="store_true", help=help_text) return parser.add_argument( *flags, type=argparse_type_from_hint(hint), help=help_text, metavar="", )""" new = """ if unwrap_union(hint) is bool: try: parser.add_argument(*flags, action="store_true", help=help_text) except Exception: pass return try: parser.add_argument( *flags, type=argparse_type_from_hint(hint), help=help_text, metavar="", ) except Exception: pass""" if old in src: f.write_text(src.replace(old, new)) print("Patched successfully!") else: print("Pattern not found - no change made") EOF
✅ Expected output

You should see: Patched successfully!

7
Run the Evaluator Test
Tests locally and registers your dataset with Fireworks

This command runs the math evaluator locally, uploads your dataset to Fireworks, and registers your API key as a secret:

Terminal
eval-protocol test

During this step you'll be asked two interactive questions:

✅ Success looks like

"1 passed" and "Created and uploaded dataset: evaluation-gsm8k-example-dataset-..."

8
Launch the RFT Job
Submit training to Fireworks cloud

Now kick off the actual reinforcement fine-tuning job. This trains Qwen3-0.6B on 1,000 GSM8K math problems for 3 epochs:

Terminal
eval-protocol create rft --base-model accounts/fireworks/models/qwen3-0p6b --epochs 3
⚠️ Model not found?

Make sure the model ID is qwen3-0p6b (using "p" for the decimal point, not "0b6"). This was a common gotcha!

💳 Cost

Qwen3-0.6B is free to tune on Fireworks — you'll see "Estimated Cost: No charge" in the dashboard.

9
Add a Payment Method
Required even for free jobs

Fireworks requires a payment method on file before it will create training jobs, even free ones. If you see this error:

Error you might see

"Error creating RFT job: Error code: 400 — payment method is required"

Go to app.fireworks.ai/settings/billing and add a card. Then re-run the command from Step 8.

✅ Job created looks like

"✅ Created Reinforcement Fine-tuning Job: accounts/xpl/reinforcementFineTuningJobs/XXXXXXXX" with dashboard links printed below.
